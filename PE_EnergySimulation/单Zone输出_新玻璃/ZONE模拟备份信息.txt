> 脚本中的“后废弃”，是指什么

  Thought for 6s (ctrl+o to expand)

● 这两个插件用 Python 的 ConstructionState 执行器做两态切换，但执行器始终没生效——窗户一直卡在默认态不切换。后来改用
  EnergyPlus 原生的 SwitchableGlazing + WindowShadingControl 方案才成功。所以这两个 Python 插件不再使用了。

✻ Cooked for 7s

> 改用原生的 SwitchableGlazing + WindowShadingControl 方案，也是用到了python脚本吗

  Thought for 8s (ctrl+o to expand)

● 不需要。SwitchableGlazing 完全在 IDF 里配置：

  idf
  Schedule:Compact,          ← 时间触发（S6）
  WindowShadingControl,      ← 切换逻辑
      SwitchableGlazing,     ← 切换类型
      PNIPAM Step 0,         ← 热态构造
      OnIfScheduleAllows,    ← 调度模式
      EC_Time_Schedule;      ← 引用上面的 Schedule

  或

  idf
  WindowShadingControl,
      SwitchableGlazing,
      PNIPAM Step 0,
      OnIfHighOutdoorAirTemperature,  ← 环境温度触发（S7）
      34.0;                          ← 阈值

  Python 只负责天气覆写（plugin_weather_only.py），窗户切换由 EnergyPlus 原生处理。

● 对。S6/S7 跑的时候，Python 脚本做的事是：

  1. 往 IDF 里插入 Schedule:Compact + WindowShadingControl 块
  2. 把 Plugin 设为 plugin_weather_only（天气覆写）
  3. 把窗构造设为 PNIPAM Step 24（基态=冷态）
  4. 启动 EnergyPlus

  运行时只有 plugin_weather_only.py 在工作——覆写天气。窗户的冷热切换是 EnergyPlus 读到 WindowShadingControl
  后自己完成的，零 Python 参与。
